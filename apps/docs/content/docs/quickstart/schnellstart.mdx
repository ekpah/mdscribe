description: Ein kompletter Minimalablauf fuer einen Entlassungsbrief mit Template, Eingabe und Output-Check.
---

## Ziel

In einem Durchlauf ein reproduzierbares Entlassungsbrief-Setup bauen, das Sie fuer weitere Faelle kopieren koennen.

## 1. Template anlegen

Nutzen Sie als Startpunkt:

```markdown
# Entlassungsbrief

## Aufnahme
<!-- Kurz, faktenbasiert, ohne Spekulation -->
{% info "aufnahmeanlass" /%}

## Diagnosen
<!-- Hauptdiagnose zuerst, dann Nebendiagnosen -->
{% info "diagnosen" /%}

## Verlauf
<!-- Chronologisch, nur relevante Ereignisse -->
{% info "verlauf" /%}

## Medikation bei Entlassung
{% info "entlassmedikation" /%}

## Procedere
{% info "procedere" /%}
```

## 2. KI-Eingabe vorbereiten

Gute Eingaben sind strukturiert und knapp:

```text
Aufnahme wegen Dyspnoe und Thoraxschmerz seit 2 Tagen.
Vorgeschichte: arterielle Hypertonie, DM2.
Befunde: Troponin erhoeht, ST-Senkungen V4-V6.
Verlauf: Monitoring auf IMC, antithrombotische Therapie, klinische Stabilisierung.
Entlassung in stabilem Zustand.
```

## 3. Generieren und nachschaerfen

1. Dokument generieren.
2. Ergebnis gegen den Eingabetext gegenpruefen.
3. Fehlende Punkte als kurze Nachinformation ergaenzen.
4. Erneut generieren.

<Callout type="tip" title="Schneller Qualitaetsgewinn">
  Fuer wiederkehrende Dokumenttypen lohnt sich ein Satz KI-Kommentare im Template (HTML-Kommentare), z. B. fuer bevorzugte Reihenfolge oder Formulierungsstil.
</Callout>

## 4. Freigabecheck vor Copy/Paste

- Diagnosen korrekt und konsistent?
- Datum/Medikation plausibel?
- Keine personenbezogenen Details enthalten, die im Cloud-Betrieb nicht eingegeben werden sollten?
- Stil passt zu Ihrem Zielsystem?

## Naechster Schritt

Wenn das klappt, bauen Sie eine zweite Variante fuer einen anderen Dokumenttyp und halten beide als Team-Standard fest.
