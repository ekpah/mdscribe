# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in MDScribe, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email us at: **support@mdscribe.de**

Include the following in your report:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (if applicable)

## Response Timeline

- We will acknowledge your report within **48 hours**
- We aim to provide a fix or mitigation plan within **7 days**
- We will coordinate disclosure with you once a fix is available

## Scope

This policy applies to the MDScribe codebase and its official deployments. Third-party dependencies should be reported to their respective maintainers.

## Supported Versions

We provide security updates for the latest release on the `main` branch.

## Acknowledgments

We appreciate the security research community's efforts in helping keep MDScribe and its users safe. Reporters will be credited in release notes unless they prefer to remain anonymous.
